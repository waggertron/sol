# Core Hypothesis Source Map

This map links the working user-model hypothesis to imported research sources.
It is a retrieval aid, not a claim that every construct is ready for product
use.

## Core Hypothesis

A useful user model may combine:

- stable trait tendencies
- context-specific states and modes
- communication and linguistic markers
- aesthetic and creative preferences
- motivational patterns
- values and goals
- behavioral regularities
- user-declared self-concepts
- contraindications and uncertainty markers

## Stable Trait Tendencies

Primary imported sources:

- `goldberg_1990_big_five`
- `ashton_lee_hexaco`
- `deyoung_2015_cybernetic_big_five`
- `roberts_2006_personality_change_meta`
- `soto_john_2017_bfi2`
- `deyoung_2007_big_five_aspects`
- `mcadams_pals_2006_new_big_five`

Model stance:

Use broad traits as organizing dimensions. Prefer facets, aspects, observed
behaviors, and context tags for actual generation control.

## Context-Specific States and Modes

Primary imported sources:

- `mischel_shoda_1995_caps`
- `fleeson_2001_density_distributions`
- `fleeson_jayawickreme_2015_whole_trait`
- `cervone_2005_personality_architecture`
- `rauthmann_2014_diamonds`

Model stance:

Represent user patterns as conditional and contextual. Avoid global labels when
the evidence supports only a work, creative, social, private, or stress-mode
pattern.

## Communication and Linguistic Markers

Primary imported sources:

- `pennebaker_king_1999_linguistic_styles`
- `tausczik_pennebaker_2010_liwc`
- `mairesse_2007_linguistic_cues`
- `schwartz_2013_open_vocab_language`
- `park_2015_social_media_language_personality`

Model stance:

Language is strong evidence for communication style and weaker evidence for
personality traits unless calibrated against self-report, context, and repeated
observations.

## Aesthetic and Creative Preferences

Primary imported sources:

- `feist_1998_creativity_meta`
- `rentfrow_gosling_2003_music_preferences`
- `big_five_trait_structure`

Model stance:

Aesthetic preferences should be modeled as preference and creative identity
signals before becoming personality claims. Openness-related constructs may help
organize the domain, but examples and user feedback should dominate generation
controls.

## Motivational Patterns

Primary imported sources:

- `deci_ryan_2000_goal_pursuits`
- `elliot_thrash_2002_approach_avoidance`
- `austin_vancouver_1996_goal_constructs`
- `sheldon_1999_self_concordance`
- `deyoung_2015_cybernetic_big_five`

Model stance:

Motivational atoms should distinguish autonomy, competence, relatedness,
approach/avoidance orientation, goal hierarchy, and self-concordance. These are
often better elicited directly than inferred passively.

## Values and Goals

Primary imported sources:

- `schwartz_2012_refined_values`
- `emmons_1986_personal_strivings`
- `sheldon_1999_self_concordance`
- `austin_vancouver_1996_goal_constructs`

Model stance:

Values and goals should be mostly user-declared or user-confirmed. The system
may infer weak hypotheses from repeated preferences, but value claims are
identity-relevant and should be editable.

## Behavioral Regularities

Primary imported sources:

- `epstein_1979_stability_behavior`
- `funder_1995_realistic_accuracy`
- `gosling_2002_room_with_cue`
- `kosinski_2013_private_traits_digital_records`
- `youyou_2015_computer_personality_judgments`

Model stance:

Behavioral signals can be predictive but are not automatically explanatory.
Aggregate across repeated observations, track source context, and separate
preference optimization from personality inference.

## User-Declared Self-Concepts

Primary imported sources:

- `markus_1977_self_schemata`
- `markus_nurius_1986_possible_selves`
- `mcadams_2001_life_stories`
- `mcadams_pals_2006_new_big_five`

Model stance:

Self-declared identity, possible selves, and narrative identity are first-class
profile layers. They can override or contextualize behavioral inference.

## Contraindications and Uncertainty Markers

Primary imported sources:

- `cronbach_meehl_1955_construct_validity`
- `shmueli_2010_explain_predict`
- `yarkoni_westfall_2017_prediction_explanation`
- `flake_fried_2020_measurement_schmeasurement`
- `lazer_2014_google_flu`
- `olteanu_2019_social_data_biases`
- `samuel_widiger_2008_ffm_personality_disorders`
- `kotov_2017_hitop`
- `bach_first_2018_icd11_personality`

Model stance:

Prediction, explanation, diagnosis, and generation utility are different
claims. Treat all profile atoms as provisional and source-scoped. Clinical and
psychopathology sources should inform boundaries, not user-facing diagnosis.
