# Research Plan

The research process should expand outward from the core claim:

> Can we infer useful, ethical, evidence-weighted personality/style constructs
> from consented data and use them to improve multimodal generation?

## Phase 1: Core Construct Map

Goal: establish which constructs are legitimate candidates for the model.

Read and summarize:

- Big Five / Five-Factor Model
- HEXACO
- whole trait theory
- CAPS and person-situation models
- psychometric construct validity
- language-based personality inference
- digital footprint prediction studies

Output:

- construct inventory
- risk notes
- profile atom candidates
- evidence quality grades

## Phase 2: Boundary and Safety Map

Goal: understand where personality modeling becomes clinically or ethically
risky.

Read and summarize:

- DSM-5 Alternative Model for Personality Disorders
- ICD-11 dimensional personality disorder model
- HiTOP
- RDoC
- research ethics for digital phenotyping and user profiling

Output:

- forbidden inference categories
- opt-in categories
- diagnostic-language blocklist
- safety review checklist

## Phase 3: Signal Map by Data Source

Goal: understand what each data source can and cannot support.

Data sources:

- writing samples
- chat history
- journals
- social posts
- creative artifacts
- preference ratings
- generated-output edits
- image/audio/video examples
- direct questionnaires

Output:

- signal matrix
- confidence rules
- data minimization rules
- source provenance format

## Phase 4: Generation Mapping

Goal: map profile atoms to output controls.

Modalities:

- text
- image
- audio/music
- voice
- video
- avatar
- UI/agent behavior

Output:

- modality control dictionary
- prompt templates
- evaluation rubrics
- feedback schema

## Phase 5: Evaluation Design

Goal: decide how to measure whether personalization works.

Candidate evaluation methods:

- user recognition: "this feels like me"
- forced-choice preference against generic outputs
- profile correction rate
- longitudinal trust score
- calibration against self-report
- output quality and usefulness ratings
- qualitative interview coding

Output:

- MVP evaluation protocol
- measurement validity warnings
- statistical plan for early tests

## Evidence Quality Grades

Use this simple grading system for source cards:

- `A`: meta-analysis, major review, established framework, or replicated body of evidence
- `B`: peer-reviewed empirical paper with relevant methods and limits
- `C`: credible framework or professional documentation, but not direct evidence
- `D`: speculative, indirect, or product hypothesis

## Source Card Template

Each source card should include:

- citation
- source type
- research domain
- key contribution
- model implications
- limits and cautions
- useful search terms
- evidence grade
- follow-up questions
