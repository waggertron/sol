## Key Contribution

CAPS is central for this project because it explains stable personality patterns
without assuming behavior is constant across all situations. It emphasizes
if-then behavioral signatures: if a person is in a particular kind of situation,
then particular cognitive, affective, and behavioral patterns are more likely.

## Model Implications

The profile should include context tags and conditional patterns.

Example:

- not just: "the user is direct"
- better: "in planning and work contexts, the user tends to prefer direct,
  low-hedge language"

CAPS supports a profile model that stores situation, appraisal, goal, emotion,
and behavior together instead of flattening them into static traits.

## Limits and Cautions

Inferring if-then patterns requires repeated observations across contexts. Sparse
data should produce hypotheses, not durable profile atoms.
