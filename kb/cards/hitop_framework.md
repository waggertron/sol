# Source Card: HiTOP Framework

## Citation

Hierarchical Taxonomy of Psychopathology (HiTOP) framework.
https://www.hitop-system.org/

Source registry id: `hitop_framework`

## Domain

- dimensional psychopathology
- hierarchical classification
- clinical research

## Key Contribution

HiTOP argues for hierarchical, dimensional organization of psychopathology rather
than relying only on categorical diagnoses. It is relevant because it shows how
psychological constructs can be represented at different levels of granularity.

## Model Implications

The project can use a hierarchy for non-clinical personalization:

- broad domains
- narrower facets
- observable indicators
- source-specific evidence

HiTOP also reinforces the need to avoid diagnostic labels when the system is not
performing validated clinical assessment.

## Limits and Cautions

HiTOP is about psychopathology, not consumer personalization. It should inform
boundary design and hierarchy, not become the user profile.
